# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest main branch | :white_check_mark: |

## Reporting a Vulnerability

Use **GitHub Security Advisories** (private vulnerability reporting):

1. Go to the repository's **Security** tab
2. Click **Report a vulnerability**
3. Fill in the details privately

Do not file public issues for security vulnerabilities.

## Response Process

1. Acknowledgment within 7 days
2. Triage and impact assessment
3. Fix development and testing
4. Coordinated disclosure

## Scope

- Public-safety scanner logic
- Extension runtime (when implemented)
- Storage and permissions handling
- Content script injection

## Out of Scope

- Browser/platform vulnerabilities
- Third-party dependency issues (report to upstream)
- Social engineering or phishing
